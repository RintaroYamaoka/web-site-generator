# ADR-0001: モノレポ採用とドキュメント駆動開発

## ステータス

採用（2025-03 想定）

## 背景

- デモサイト生成には「Web UI」「生成エンジン」「デプロイ」「テンプレート」の複数コンポーネントがあり、共有型定義やテンプレートを一括で管理したい。
- 仕様の変更をコードより先にドキュメントで固め、実装の一貫性を保ちたい。

## 決定

1. **モノレポ**  
   `apps/` と `packages/` で構成し、pnpm workspace（または npm/yarn workspaces）で管理する。テンプレートは `templates/` に置く。

2. **ドキュメント駆動**  
   設計の正は `docs/` に置く。要件は `requirements/`、構成は `architecture/`、詳細仕様は `specs/`、重要な判断は `decisions/`（ADR）に記載する。実装はこれらのドキュメントに従う。

## 結果

- パッケージ間の型・インターフェースを `packages/shared` で共有しやすい。
- 新規参加者や将来の自分が、`docs/` を読むだけで方針と構造を把握できる。
- 仕様変更時はドキュメントを先に更新する習慣がつく。
