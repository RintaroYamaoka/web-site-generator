# ADR-0004: 仮ページ（仕様書）生成はロジック、実装のみ LLM

## ステータス

採用

## 背景

設計図（仮ページ・仕様書）を LLM が生成し、それを別の LLM が読んで実装する構成だと、**両方の出力が非決定的**なため、ブレが二重になり、再現性や意図の保持が難しくなる。

## 問題

- 1 台目の LLM（仮ページ生成）の出力が毎回少しずつ変わる。
- 2 台目の LLM（実装）がその「毎回違う設計図」を入力とするため、解釈も毎回変わる。
- 結果として **ブレが大きい**。同一入力でも最終成果物のばらつきが増える。

## 決定

- **仮ページ（仕様書）の生成は LLM を使わず、ロジック**（テンプレートと入力スキーマの組み立て）で行う。
- **LLM を呼ぶのは実装の 1 回だけ**。入力はロジックが生成した「常に同一の仮ページ」とする。
- ユーザー入力 → 仮ページ → 実装 のうち、「ユーザー入力 → 仮ページ」はロジック、「仮ページ → 実装」は LLM。

## 結果

- 同一入力からは **常に同一の仮ページ** が得られ、ブレの原因は実装フェーズの 1 回に限定される。
- コスト・レイテンシ・再現性の面で有利。ユーザーが入力した文言がロジックでそのまま仮ページに載るため、意図のずれも起きにくい。
- 要件・パイプライン（[02-generation-pipeline.md](../specs/02-generation-pipeline.md)）および [06-llm-optimization.md](../specs/06-llm-optimization.md) にこの方針を反映する。
