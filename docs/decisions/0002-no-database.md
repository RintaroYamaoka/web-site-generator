# ADR-0002: 永続化に DB を使わないこと

## ステータス

採用（2025-03 想定）

## 背景

- 初期スコープでは「入力 → 生成 → Vercel デプロイ → URL 返却」の 1 回きりのフローで十分と見込んでいる。
- 生成履歴の永続化や複数ユーザー・テナント管理は含めないため、サーバー側の永続化層を用意する必要性が低い。

## 決定

- **データベース（永続化 DB）は導入しない。**
- 入力内容の一時保存は、ブラウザのセッションまたは localStorage 等のクライアント側保存で対応する。
- 生成結果・デプロイ URL は、デプロイ完了時にユーザーに渡し、サーバー側では保持しない。

## 結果

- インフラがシンプルになり、Vercel のサーバーレス＋静的/API のみで運用しやすい。
- 将来「生成履歴の一覧」「URL の再取得」などの要件が出た場合は、docs で要件を拡張し、その時点で DB または Vercel のストレージ等を検討する。
