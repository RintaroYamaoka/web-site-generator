# ADR-0003: 構造・中身を先に定義し、デザインは後から決める

## ステータス

採用

## 背景

Web サイト制作を自動化するにあたり、「何を先に決めるか」の順序が成果物の質に直結する。AI（LLM）は入力として与えられたコードやテキストをコンテキストとして強く参照し、それに沿った出力をしがちである。

## 問題

- 先にデザイン（見た目・レイアウト・コンポーネント）を実装してから中身を決めると、LLM はそのデザインのコードをコンテキストとして読み、**サイトの目的やコンテンツよりデザインに引きずられた実装**になりやすい。
- その結果、**中身のイメージとデザインが最適でない**サイトができがちである。

## 決定

- **構造と中身を先に定義し、その後にデザインを決める**ことを基本方針とする。
- 本プロジェクトのフローでは、ユーザー入力から **マークダウン形式の仮ページ（ページ構成・見出し・本文・必須文言）** をまず生成する。その「中身」だけを入力として、**後段で LLM がデザイン・UI を実装**する。デザインは中身に合わせて決める。

## 結果

- 仮ページが「正」となり、実装フェーズでは「この内容をどう見せるか」だけに集中できる。
- 要件・概要（[00-overview.md](../requirements/00-overview.md)）にこの方針を明記し、パイプライン（仮ページ → 実装）の順序を変えない。
- **LLM 最適化**: 各フェーズでコンテキストに含める／含めないものを厳密にし、デザインコードを先に渡さない設計は [specs/06-llm-optimization.md](../specs/06-llm-optimization.md) に記載する。
