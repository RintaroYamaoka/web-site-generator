# ディレクトリ構造

本プロジェクトのルートおよび主要ディレクトリの役割を定義する。実装時はこの構造に従い、変更時は本ドキュメントを先に更新する。

## ルート構成（方針）

- **docs/** … ドキュメント駆動開発用の設計・要件・仕様（コードの正とする）
- **apps/** … 実行可能なアプリケーション（Web UI、CLI、API 等）
- **packages/** … 共有ライブラリ（サイト生成エンジン、Vercel 連携等）
- **templates/** … 将来のテンプレートベース生成用。現行フローでは LLM がマークダウンから Next.js を一から生成するため、`next-demo` は未使用。
- **scripts/** … ビルド・デプロイ・検証用のスクリプト

```
web-site-generator/
├── docs/                    # ドキュメント（設計の正）
│   ├── README.md            # ドキュメント駆動の進め方・索引
│   ├── requirements/        # 要件・ユーザーストーリー
│   ├── architecture/        # アーキテクチャ・ディレクトリ設計
│   ├── specs/               # 詳細仕様（API・UI・パイプライン）
│   ├── decisions/           # ADR（アーキテクチャ決定記録）
│   └── templates/           # テンプレート設計メモ・コンテンツ方針
├── apps/
│   ├── web/                 # ユーザー向け Web UI（入力・生成トリガー・URL 表示）
│   └── api/                 # （必要なら）生成ジョブ・デプロイ制御用 API
├── packages/
│   ├── core/                # サイト生成ロジック（ページ構成→コンポーネント・ファイル）
│   ├── deploy/              # Vercel デプロイ連携
│   └── shared/              # 共通型・ユーティリティ
├── templates/
│   └── next-demo/           # デモ用 Next.js テンプレート（App Router 想定）
├── scripts/                 # デプロイ・検証・ビルド用スクリプト
├── package.json             # ルート（モノレポ用）
├── pnpm-workspace.yaml      # または npm/yarn workspaces
└── README.md
```

## 各ディレクトリの責務

### docs/

| パス | 役割 |
|------|------|
| `requirements/` | 目標・ユーザーストーリー・受け入れ条件 |
| `architecture/` | システム構成・ディレクトリ・データフロー |
| `specs/` | API・入力スキーマ・生成パイプラインの詳細 |
| `decisions/` | 技術選定・設計判断の ADR |
| `templates/` | テンプレートのページ構成・デザイン方針のメモ |

### apps/

| パス | 役割 |
|------|------|
| `web/` | Next.js または React の SPA/SSR。ユーザーが「内容・ページ構成・イメージ」を入力し、生成をトリガーし、デプロイ済み URL を表示する |
| `api/` | （オプション）長時間の生成・デプロイをジョブとして実行し、ステータス・URL を返す API |

### packages/

| パス | 役割 |
|------|------|
| `core/` | ユーザー入力から **ロジック**（マークダウン組み立て用テンプレート＋入力）で仮ページ（仕様書）を生成。仮ページを **LLM（1 回のみ）** で読み、Next.js を実装（コード生成）。 |
| `deploy/` | 生成成果物の Git リポジトリ初期化（必要に応じてリモート作成）、Vercel へのデプロイ（API/CLI） |
| `shared/` | 入力スキーマ型、設定型、ログ等の共通コード |

### templates/

| パス | 役割 |
|------|------|
| `next-demo/` | 将来のテンプレートベース生成用の Next.js 雛形。**現行フローでは未使用**（LLM がマークダウンから一から生成するため）。 |

### scripts/

- 一括ビルド・デプロイ・E2E 検証など、ルートから実行するスクリプトを配置する。

## ドキュメント駆動での運用

- **新規ディレクトリ・パッケージを追加する場合**  
  まず本ファイル（`docs/architecture/01-directory-structure.md`）に追記し、理由は必要に応じて `docs/decisions/` に ADR として残す。
- **役割が変わった場合**  
  上記テーブルとツリーを更新し、関連する `specs/` を整合させる。
