# プロジェクト概要・目標

## プロジェクト名

**Web サイト制作代行デモサイト自動生成システム**（web-site-generator）

## 背景

Web サイト制作代行において、顧客への提案用・デモ用サイトを手作業で用意する工数が大きい。  
内容やページ構成が「ざっくり」決まっている段階でも、すぐにデモ URL を共有できるようにしたい。

## 基本方針：構造・中身を先に、デザインは後から

**サイトの構造と中身を先に定義し、その後にデザインを決める**ことを基本とする。

- AI（LLM）はコードや既存の実装をコンテキストとして強く参照する。先にデザインを実装すると、そのコードに引きずられ、**サイトの中身（コンテンツ・目的）とデザインが噛み合わない**ことが多い。
- そこで本プロジェクトでは、**マークダウン形式の仮ページ（構造・文言・セクション）を先に生成**し、その「中身」を入力として **後からデザイン・UI を実装**するフローを採用する。中身が正とする。

## 目標（達成すべきこと）

1. **仮ページ駆動のデモサイト生成**  
   ユーザーが「ページ数（1〜5）」「何のサイトか」「各ページのセクション内容」「必ず入れてほしい文言」を入力すると、**LLM がマークダウン形式の仮ページ（各ページの内容がそのまま並んだドラフト）を生成**する。その仮ページを LLM が読み込み、**イメージに合った最新技術の UI** で実装する。

2. **リポジトリ作成と自動デプロイ**  
   実装成果物で **Git リポジトリを作成**し、**Vercel にデプロイ**する。ユーザーは **生成された URL** からデモサイトを閲覧できる。

3. **ユーザー体験**  
   ユーザーは入力から、仕様書 → 実装 → デプロイまで一連のフローを経て、デプロイ済み URL を取得する。

## スコープ（含めるもの・含めないもの）

- **含める**: 入力 UI（ページ数 1〜5・サイト説明・各ページのセクション内容・必須文言）、マークダウン形式の仮ページの LLM 生成、仮ページを読んだ LLM による実装、Git リポジトリ作成、Vercel デプロイ連携、生成 URL の返却
- **含めない（初期）**: 決済、会員認証、複数テナント、既存 CMS 連携、**データベース（永続化 DB）**（必要になったら docs で拡張）
- 入力・生成結果の保持はセッションまたはクライアント側の一時保存で対応する。

## 成功基準

- ユーザーが入力から 10 分以内にデプロイ済み URL を取得できる
- 生成サイトはモバイル対応・アクセシビリティを考慮した構成とする
- 同一入力に対して再現可能な生成結果が得られる（シード等で制御可能とする方針を検討）
