# ユーザーストーリー・受け入れ条件

## エピック: デモサイトの生成から公開まで

### US-1: サイト内容の入力

**ストーリー**  
制作担当者として、デモに使う「ページ数」「各ページのセクション見出し・セクション数・セクション内容（おまかせ可）」「セクションごとの画像（添付 or テキスト指定 or おまかせ）」「必須文言」「デザイン希望（おまかせ可）」を入力したい。そうすれば、その内容を元に 1 本の仕様書（ページ構成＋仮ページ）がロジックで生成され、LLM がサイトを実装する。

**受け入れ条件**

- [ ] ページ数を **1 から 5 の範囲**で選択できる
- [ ] 各ページの **セクション見出し・セクション数・セクション内容** を入力できる（内容はおまかせの場合は固定テキストが仕様書に入る）
- [ ] 各セクションの **画像** を「添付」「テキストで指定（後で生成可）」「おまかせ」のいずれかで指定できる
- [ ] 必ず入れてほしい文言（会社名・キャッチコピー・法的表記など）を入力できる
- [ ] **デザイン希望** を入力できる（おまかせあり）
- [ ] 入力内容は一時保存でき、後から編集できる（ブラウザのセッションまたは localStorage 等のクライアント側保存で対応。サーバー側 DB は使わない）

---

### US-2: 1 本のマークダウン（仕様＋仮ページ）の生成

**ストーリー**  
入力内容を送信すると、**ロジック**（テンプレートと入力スキーマの組み立て）で **ページ構成の仕様** と **仮ページ** を **1 本のマークダウンファイル** にまとめた仕様書が生成される。LLM は使わない。

**受け入れ条件**

- [ ] ユーザー入力に基づき、**ロジック**で「ページ構成の仕様」と「仮ページ」が 1 本のマークダウンにまとめられる
- [ ] 内容・画像のおまかせは事前に決めた固定テキストで仕様書に記載される
- [ ] 同一入力なら同一の 1 本マークダウンが得られる（再現可能）
- [ ] 生成された仕様書は後段の LLM が読み、サイトのソースコード生成の入力として使える
- [ ] 生成された仕様書はユーザーが確認できる（表示またはダウンロード）

---

### US-3: 仕様書に基づく実装

**ストーリー**  
1 本のマークダウン（仕様＋仮ページ）を LLM が読み込み、Next.js / Tailwind / Vercel に固定された技術仕様に従って Web サイトのソースコードを生成する。

**受け入れ条件**

- [ ] 仕様書の内容・構成に沿った Next.js プロジェクト（App Router、TypeScript、Tailwind）が生成される
- [ ] 生成サイトの技術仕様（Next.js / Tailwind / Vercel）に固定されている
- [ ] 必須文言が仕様書どおりに含まれる
- [ ] 生成結果は Vercel でビルド可能な成果物として出力される

---

### US-4: Git リポジトリの作成

**ストーリー**  
実装された Next.js プロジェクトで Git リポジトリを作成し、デプロイに備える。

**受け入れ条件**

- [ ] 生成成果物を Git リポジトリとして初期化（コミット）できる
- [ ] リポジトリは Vercel デプロイで参照可能な形で用意される（ローカル／リモートは別仕様で定義）

---

### US-5: Vercel への自動デプロイ

**ストーリー**  
Git リポジトリにしたサイトを Vercel に自動デプロイし、本番 URL を取得したい。

**受け入れ条件**

- [ ] デプロイは Vercel API または CLI を用いて自動実行される
- [ ] デプロイ完了後に **生成された URL** がユーザーに表示される
- [ ] デプロイ失敗時はエラー内容が分かる形で通知される

---

### US-6: 生成 URL での閲覧

**ストーリー**  
デプロイ完了後、渡された URL からデモサイトを誰でも閲覧できる。

**受け入れ条件**

- [ ] URL は HTTPS で提供される
- [ ] モバイル・デスクトップの両方で表示が崩れない
- [ ] アクセシビリティの基本（セマンティック HTML・コントラスト等）を満たす方針でテンプレートが設計されている
