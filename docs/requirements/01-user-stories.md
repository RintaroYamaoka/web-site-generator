# ユーザーストーリー・受け入れ条件

## エピック: デモサイトの生成から公開まで

### US-1: サイト内容の入力

**ストーリー**  
制作担当者として、デモに使う「ページ数」「何のサイトか」「各ページのセクション内容」「必ず入れてほしい文言」を入力したい。そうすれば、その内容を元にマークダウン形式の仮ページ（ドラフト）が生成され、さらにサイトが実装される。

**受け入れ条件**

- [ ] ページ数を **1 から 5 の範囲**で選択できる
- [ ] 「何のサイトか」（サイトの目的・業種・対象者など）をテキストで入力できる
- [ ] 各ページのセクション内容（見出し・本文の意図・構成）を入力できる
- [ ] 必ず入れてほしい文言（会社名・キャッチコピー・法的表記など）を入力できる
- [ ] 入力内容は一時保存でき、後から編集できる（ブラウザのセッションまたは localStorage 等のクライアント側保存で対応。サーバー側 DB は使わない）

---

### US-2: マークダウン形式の仮ページの生成

**ストーリー**  
入力内容を送信すると、LLM がその内容を元に **マークダウンで書かれた仮ページ（ドラフト）** を生成する。各ページの見出し・本文がそのままマークダウンで並んだイメージである。

**受け入れ条件**

- [ ] ユーザー入力に基づき、各ページの内容がマークダウンで記述された「仮ページ」が生成される（本番に載せる文言・構成がそのまま読める形）
- [ ] 仮ページは後段の「実装」ステップで LLM が読み込み、同じ内容・構成を UI で実装する入力として使える
- [ ] 生成された仮ページはユーザーが確認できる（表示またはダウンロード）

---

### US-3: 仮ページに基づく実装

**ストーリー**  
マークダウンで書かれた仮ページを LLM が読み込み、その内容・構成をイメージに合った最新技術の UI で Web サイトとして実装する。

**受け入れ条件**

- [ ] 仮ページの内容・構成に沿った Next.js プロジェクト（ルート・コンポーネント・スタイル）が生成される
- [ ] 最新の技術・UI トレンドを反映した実装方針が取られる
- [ ] 必須文言が仮ページどおりに含まれる
- [ ] 生成結果はビルド可能な Next.js の成果物として出力される

---

### US-4: Git リポジトリの作成

**ストーリー**  
実装された Next.js プロジェクトで Git リポジトリを作成し、デプロイに備える。

**受け入れ条件**

- [ ] 生成成果物を Git リポジトリとして初期化（コミット）できる
- [ ] リポジトリは Vercel デプロイで参照可能な形で用意される（ローカル／リモートは別仕様で定義）

---

### US-5: Vercel への自動デプロイ

**ストーリー**  
Git リポジトリにしたサイトを Vercel に自動デプロイし、本番 URL を取得したい。

**受け入れ条件**

- [ ] デプロイは Vercel API または CLI を用いて自動実行される
- [ ] デプロイ完了後に **生成された URL** がユーザーに表示される
- [ ] デプロイ失敗時はエラー内容が分かる形で通知される

---

### US-6: 生成 URL での閲覧

**ストーリー**  
デプロイ完了後、渡された URL からデモサイトを誰でも閲覧できる。

**受け入れ条件**

- [ ] URL は HTTPS で提供される
- [ ] モバイル・デスクトップの両方で表示が崩れない
- [ ] アクセシビリティの基本（セマンティック HTML・コントラスト等）を満たす方針でテンプレートが設計されている
