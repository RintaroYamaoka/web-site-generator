# 生成パイプライン仕様

ユーザー入力 → **マークダウン形式の仮ページの生成（LLM）** → **仮ページを読んだ LLM による実装** → **Git リポジトリ作成** までを定義する。デプロイは [Vercel デプロイ仕様](./03-vercel-deploy.md) に従う。

## 入力・出力

- **入力**: [入力スキーマ](./01-input-schema.md) に準拠した `SiteGenerationInput`（ページ数 1〜5・サイト説明・各ページのセクション・必須文言）
- **中間成果物**: [マークダウン形式の仮ページ](./04-markdown-spec-format.md)（各ページの見出し・本文がそのまま並んだ 1 つの Markdown ドキュメント）
- **最終成果物**: ビルド可能な Next.js プロジェクトが入った 1 つのディレクトリ。かつ Git リポジトリとして初期化済み（必要に応じてリモート作成済み）

## パイプラインのステップ

### フェーズ 1: マークダウン形式の仮ページの生成

1. **入力の検証**  
   `SiteGenerationInput` をバリデーションし、不正ならエラーを返して終了。

2. **LLM による仮ページ生成**  
   ユーザー入力をプロンプト（または構造化メッセージ）に組み込み、LLM を呼び出す。  
   - 出力は [04-markdown-spec-format.md](./04-markdown-spec-format.md) に沿ったマークダウンとする。  
   - 各ページの内容（見出し・本文・必須文言）が、本番に載せる形でそのまま並んだ「仮ページ」のイメージで生成する。  
   - **LLM 最適化**: コンテキストには **デザイン・既存コードを含めない**。ユーザー入力とフォーマット説明のみ。詳細は [06-llm-optimization.md](./06-llm-optimization.md)。

3. **仮ページの保存・返却**  
   生成されたマークダウンをファイルとして保存するか、メモリ上で保持する。  
   - ユーザーへの表示・ダウンロード用と、後段の「実装」ステップへの入力用の両方に使う。

### フェーズ 2: 仮ページに基づく実装

4. **LLM による実装**  
   マークダウンで書かれた仮ページを LLM に入力として渡し、**同じ内容・構成をイメージに合った最新技術の UI** で Next.js プロジェクトとして実装させる。  
   - 出力はファイル群（`package.json`, `app/`, `components/`, スタイル等）。  
   - App Router を前提とし、仮ページに書かれた見出し・本文・文言をすべて反映する。  
   - 実装手段は「LLM がコードを生成し、それをファイルとして書き出す」形を想定する。  
   - **LLM 最適化**: コンテキストは **仮ページ（マークダウン）と技術要件の短文のみ**。既存のデザインコード・テンプレートは渡さない。プロンプトで「デザインは中身に合わせて決める」と明示する。詳細は [06-llm-optimization.md](./06-llm-optimization.md)。

5. **ビルドの可否確認**  
   生成ディレクトリで `next build`（または `pnpm build`）を実行し、成功することを確認する。失敗した場合はエラーを返す（オプション: リトライや LLM へのフィードバックは別仕様で検討）。

6. **出力ディレクトリの返却**  
   生成された Next.js プロジェクトのディレクトリパスを、呼び出し元に返す。

### フェーズ 3: Git リポジトリの作成

7. **Git 初期化**  
   生成ディレクトリで `git init` を実行し、全ファイルをコミットする。

8. **リモートの扱い（オプション）**  
   Vercel が Git 連携でデプロイする場合は、リモートリポジトリ（GitHub/GitLab/Bitbucket 等）を作成し、push する。  
   ZIP アップロードでデプロイする場合は、ローカル Git のみで完了とする。  
   どちらを採用するかは [03-vercel-deploy.md](./03-vercel-deploy.md) および実装方針に合わせる。

## コンポーネントの責務

- **仮ページ生成**: `packages/core` または API（LLM 呼び出し）が担当。
- **実装生成**: 同上。仮ページ（マークダウン）を LLM に渡し、生成されたコードをファイルとして書き出す。
- **Git 操作**: `packages/deploy` または専用パッケージが、生成ディレクトリに対して `git init` と `git add` / `git commit` を実行。必要ならリモート作成・push。

## 冪等性・再現性

- 同じ `SiteGenerationInput` と同一の LLM 設定からは、**仮ページ**は可能な限り再現可能にする（プロンプトを固定化する）。  
- **実装**は LLM の非決定的な振る舞いの影響を受けるため、完全な再現は保証しない。必要に応じてシードや温度パラメータで制御する。

## LLM の性質に最適化した設計
