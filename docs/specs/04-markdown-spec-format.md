# マークダウン形式の仕様書（1 本の構成）

**ロジック**が入力スキーマから生成する **唯一の成果物** は、**1 本のマークダウンファイル** である。その中に **ページ構成の仕様** と **仮ページ** の両方を含む。LLM はこの 1 本だけを読んでサイトのソースコードを生成する。

## 1 本のマークダウンに含めるもの

1. **ページ構成の仕様**（前半）
   - ページ一覧（パス・名前・セクション数）、デザイン希望、必須文言一覧。
   - 画像の扱い（添付 ref / テキスト指定 / おまかせ）を一覧またはページ・セクションごとに記載。
2. **仮ページ**（後半）
   - ページごとに見出し・本文・画像プレースホルダーをマークダウンで記述。
   - 内容おまかせのセクションには [固定文言の正式定義](#固定文言の正式定義) の「セクション本文おまかせ」を入れる。
   - 画像は ref（例: `![ref:image_1]`）または [固定文言の正式定義](#固定文言の正式定義) の「画像おまかせ」／テキスト指定（`画像: 〇〇を表示`）で表す。

## 画像の表し方（ロジック→LLM の契約）

| ユーザー指定 | マークダウン上の表記 |
|--------------|----------------------|
| 添付 | `![ref:image_1](添付画像1)` など、ref で参照。LLM 入力時に実体を渡す。 |
| テキストで指定 | `画像: 〇〇を表示` のように説明をそのまま記載。後で生成する場合はその旨を記載可。 |
| おまかせ | [固定文言の正式定義](#固定文言の正式定義) の「画像おまかせ」を使用。 |

ロジックは画像の実体を生成しない。ref または文言だけを仕様に書く。

## 固定文言の正式定義

おまかせ指定時にマークダウンへ出力する文字列は、以下を **正式な値** とする。ロジック・プロンプト・テストはこの値に統一する。

| 種別 | 正式な文字列 | 用途 |
|------|--------------|------|
| セクション本文おまかせ | `AIにおまかせ` | 内容を LLM に任せるセクションの本文 |
| 画像おまかせ | `画像: おまかせ` | 画像を LLM に任せるセクションの画像プレースホルダー |

## 構成例（骨子）

```markdown
# ページ構成仕様

## ページ一覧
| パス | 名前 | セクション数 |
|------|------|-------------|
| /    | トップ | 3 |
| /about | 会社概要 | 2 |

## デザイン希望
ミニマル

## 必須文言
- 会社名: 株式会社サンプル
- キャッチコピー: 〇〇を、もっと簡単に。

## 画像指定
- トップ セクション2: 添付 (ref: image_1)
- トップ セクション3: おまかせ

---

# 仮ページ

# トップ (/)
## 株式会社サンプル
〇〇を、もっと簡単に。...

## 強み
[ref:image_1] を表示

## 実績
画像: おまかせ

# 会社概要 (/about)
...
```

## 記載ルール

- 本番に載せる文言はそのまま書く。おまかせの本文・画像は [固定文言の正式定義](#固定文言の正式定義) に置換する。
- 見出しレベル（`#`, `##`）はページ・セクションの階層を反映する。
- 1 本のマークダウンで「仕様」と「仮ページ」の両方を LLM が読める形にしておく。

## 実装側の解釈

- LLM はこの 1 本のマークダウンを入力とし、[07 生成サイト技術仕様](./07-generated-site-tech-spec.md)（Next.js / Tailwind / Vercel）に従ったソースコードを生成する。
- 画像 ref は、LLM に渡すときに添付ファイルや URL と紐付ける。`画像: おまかせ` はプレースホルダーまたは後段の画像生成で対応する。
