# マークダウン形式の仮ページ（仕様書）

LLM がユーザー入力から生成する **最初の成果物** は、**マークダウンで書かれた仮ページ** のイメージとする。  
「仕様書」というより、**サイトの各ページの内容がそのままマークダウンで並んだドラフト**であり、見出し・本文・リストなどが本番のページ構成と対応している。後段の実装 LLM は、この仮ページを読んで「この見た目・内容を最新技術の UI で実装する」と解釈する。

## イメージ

- 各ページの **実際の文言・見出し・段落** がマークダウンで書かれている。
- 読むと「サイトをマークダウンでプレビューしたような」一つのドキュメントになる。
- 必須文言（会社名・キャッチコピーなど）も、仮ページ内の該当位置にそのまま記載する。

## 構成

1. **先頭（任意）**  
   - サイトのトーン・対象者・技術方針などを 1 〜 2 行でメモしてもよい。実装時の参考用。  
   - 例: `<!-- トーン: フォーマル / 対象: 法人 / Next.js App Router -->`

2. **ページごとの仮ページ**  
   - 各ページを **見出しで区切る**。見出しにページ名とパスを明示する。  
   - 例: `# トップ (/)` の直下に、そのページに表示する内容をマークダウンで記述する。  
     - ヒーローならキャッチコピーやリード文をそのまま書く。  
     - セクション見出しは `## 〇〇`、本文はその下の段落・リストで書く。  
   - 次のページは同じ要領で `# 会社概要 (/about)` のように区切り、その下にそのページの見出し・本文を書く。

3. **区切り**  
   - ページとページの間は、見出し `# ページ名 (パス)` で区切れば十分。必要なら `---` で区切ってもよい。

## 記載ルール

- **仮ページ** なので、**本番に載せる（または載せたい）文言をそのまま書く**。  
  「ここに会社名を入れる」ではなく、実際の「株式会社サンプル」などを書く。
- 見出しレベル（`#`, `##`, `###`）は、そのページ内の階層を反映する。
- リスト・強調・リンクなど、マークダウンの一般的な記法でよい。実装側はこれを「この内容を UI コンポーネントで再現する」入力とする。

## 例（仮ページのイメージ）

```markdown
<!-- 法人向け製品紹介。トーン: フォーマル。 -->

# トップ (/)

## 株式会社サンプル
〇〇を、もっと簡単に。業務効率を変える、クラウドサービスです。

## 3つの強み

- **スピード** — 即日利用開始
- **安心** — 国内データセンター
- **サポート** — 専任担当がつきます

## お問い合わせ
まずはお気軽にご相談ください。[お問い合わせページ](/contact)へ。

---

# 会社概要 (/about)

## 会社概要
私たちは〇〇年創業の...

## 沿革
- 〇〇年 設立
- 〇〇年 〇〇サービス開始
```

## 実装側の解釈

- 実装 LLM は、このマークダウンを「**各ページの内容と構造が定義された仮ページ**」として読み、Next.js で同じ内容・構成を最新技術の UI（コンポーネント・スタイル・レイアウト）で実装する。
- 見出し・段落・リストは、セマンティックな HTML とコンポーネントに変換する。デザインやレイアウトは仕様書（仮ページ）の意図を保ちつつ、現代的でアクセシブルな形で実装する。
