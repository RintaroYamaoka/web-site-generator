# 生成サイトの技術仕様（固定）

LLM が出力する **生成サイト** は、以下の技術スタックに **固定** する。プロンプトや評価時はこの仕様に従う。

## 固定項目

| 項目 | 固定内容 | 備考 |
|------|----------|------|
| **フレームワーク** | Next.js | App Router を使用する。 |
| **スタイル** | Tailwind CSS | 生成コードのスタイルは Tailwind で書く。 |
| **デプロイ先** | Vercel | ビルド・デプロイは Vercel を前提とする。 |
| **言語** | TypeScript | `.tsx` / `.ts` を使用する。 |
| **Node** | 18.x 以上 | Vercel のデフォルトに合わせる。 |
| **パッケージマネージャ** | npm | 生成プロジェクトの `package.json` と lock は npm で統一する（Vercel がそのままビルドできるようにする）。 |
| **画像** | next/image | 画像表示は `next/image` を使用する。 |

## Next.js の前提

- **App Router** を使用する。`app/` 配下に `layout.tsx`、`page.tsx` を配置する。
- ルートは仕様書のページ構成（パス）に合わせる。例: `/`, `/about`, `/services`。
- メタデータは `metadata`  export または `generateMetadata` で設定する。

## Tailwind CSS

- `tailwind.config.ts`（または `.js`）を用意する。必要に応じてテーマ拡張。
- デザイン希望（ミニマル・コーポレート・おまかせ等）は、Tailwind のクラス選びやコンポーネント構成で反映する。

## Vercel

- `next build` でビルド可能であること。Vercel はデフォルトで `next build` を実行する。
- 環境変数が不要な静的〜ハイブリッド構成を前提とする（必要なら別途仕様で定義）。

## その他固定すべきと判断したもの

- **ESLint**: Next.js の推奨設定（`eslint-config-next`）を前提とする。

## LLM が選ぶもの（固定しない）

**アイコン・フォント・コンポーネントパターン・その他 UI 要素は固定しない。** LLM が **ページ構成とデザイン希望に合わせて、トレンドに合ったもの** を選ぶ。

- **アイコン**: 仕様で指定がなければ、LLM がページ構成・セクションの用途に合うライブラリやアイコンを選ぶ（例: Heroicons, Lucide, Phosphor 等）。トレンドを考慮する。
- **フォント**: デザイン希望やトーンに合わせて LLM が選ぶ（`next/font` 利用可）。
- **レイアウト・余白・コンポーネントの形**: デザイン希望と仕様書の内容に合わせて、LLM が現代的でトレンドに沿った UI を選ぶ。

プロンプトでは「アイコン・フォント・その他 UI は、ページ構成とデザイン希望に合わせてトレンドのものを選ぶこと」と明示する。選定の一貫性が必要な場合は [08 LLM 選定](./08-llm-selection.md) で利用モデルや方針を決めたうえで、プロンプトで誘導する。

## 固定しないもの（仕様書で変えてよいもの）

- デザイン希望（ミニマル・コーポレート・おまかせ等）に応じたレイアウト・色・余白。
- ページ数・セクション数・文言・画像の有無。これらはすべて仕様書（1 本のマークダウン）で決める。
- 上記「LLM が選ぶもの」に挙げたアイコン・フォント・UI パターン。

## LLM への指示

実装フェーズのプロンプトでは、「生成するサイトは Next.js（App Router）・Tailwind CSS・TypeScript を用い、Vercel にデプロイすることを前提とする」と明示する。本ドキュメントの固定項目を要約してプロンプトに含めてもよい。
