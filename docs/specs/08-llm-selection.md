# LLM 選定

本プロジェクトでは **実装フェーズで 1 回だけ** LLM を呼ぶ。**実装を始める前に、利用する LLM（プロバイダ・モデル）を決めておく**。ここに選定結果と選定理由を記載する。

## なぜ先に決めるか

- プロンプト設計・コンテキスト長・出力形式（ストリーミング／一括）はモデルによって変わる。
- コスト・レイテンシ・利用規約もプロバイダ・モデルで異なる。スコープと予算に合わせて選ぶ必要がある。
- アイコン・フォント・UI など「トレンドに合わせて LLM が選ぶ」部分も、モデルの能力（コード生成・最新トレンドの理解）に依存する。

## 選定時に決めること

| 項目 | 内容 |
|------|------|
| **プロバイダ** | 例: OpenAI, Anthropic, Google, その他（API またはセルフホスト） |
| **モデル** | 例: GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro 等。コード生成・長文コンテキストに対応したものを選ぶ。 |
| **API 形態** | REST / SDK。ストリーミングするか、一括でファイル群を受け取るか。 |
| **コンテキスト長** | 1 本のマークダウン＋技術仕様＋出力コードが収まる長さを確保する。 |
| **利用規約・コスト** | 生成コードの利用条件、入力・出力の扱い。トークン単価と想定呼び出し回数。 |

## 選定基準の例

- **コード生成品質**: Next.js / TypeScript / Tailwind の一般的なパターンを正しく出力できるか。
- **コンテキスト長**: 仕様書（1 本のマークダウン）が長くなっても、全体を入力に収められるか。
- **出力形式**: 複数ファイルを一括で返せるか、またはチャンクで返してクライアントで組み立てるか。
- **トレンド**: アイコン・UI パターンについて「トレンドに合わせて選ぶ」指示にどれだけ従えるか（学習データの鮮度に依存）。
- **コスト・速度**: 1 サイトあたりの想定コストとレイテンシが許容範囲か。

## 選定結果（実装前に記載する）

- **プロバイダ**: **Google**
- **モデル**: **Gemini 3 Flash Preview**（`gemini-3-flash-preview`）
- **選定理由・備考**: 無料枠で使えるうち、コード生成向けに推奨される最新モデル。1M トークンコンテキストで 1 本マークダウン＋技術仕様＋出力コードを収められる。API は [Google AI Studio](https://aistudio.google.com/) で API キーを取得し、Node では **`@google/genai`**（新 SDK）を使用。環境変数 `GEMINI_API_KEY` でキーを渡す。

選定したら本セクションを更新し、`packages/core` の実装ではこのモデル・API を前提とする。変更する場合は本ドキュメントと関連 ADR を更新する。

## Gemini を選んだ場合の参考（無料枠・レート制限）

- **Free ティア**: 対象国では無料で入力・出力トークンを使用可能。利用可能なモデルは限定（Pro 系は無料枠なしのことがある。Flash 系は無料で使えるモデルあり）。
- **レート制限**: プロジェクトごとに **RPM（分あたりリクエスト数）・TPM（分あたりトークン数）・RPD（1日あたりリクエスト数）** が適用される。実際の数値は [AI Studio のレート制限ページ](https://aistudio.google.com/rate-limit) で確認する。
- **本プロジェクトとの関係**: 1 サイト生成 = **LLM を 1 回だけ**呼ぶため、RPD が実質の上限。無料枠の RPD が例えば 15〜20 の場合、1 日あたり十数サイト程度までが目安。開発・デモ用途では無料枠で足りる可能性が高い。
- **有料化**: 請求を有効にすると Tier 1 となり、レート制限が緩和され、Pro 系など全モデルが利用可能。料金は [Gemini API 料金](https://ai.google.dev/gemini-api/docs/pricing) を参照。
- **無料枠超過時**: 超過分が**自動で請求されることはない**。無料枠を超えると API がエラーを返して呼び出しが止まるだけ。課金が発生するのは、請求先アカウントと支払い方法を**自分で設定した場合のみ**。
